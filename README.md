# 한희경팀 AI팀 — 담당자별 작업 지침 사이트

AI강사 한희경의 AI팀 9인 시스템을 누구나 클릭만으로 사용할 수 있는 웹사이트입니다.

🔗 **배포 후 주소**: `https://[GitHub아이디].github.io/[저장소이름]/`

---

## 무엇을 할 수 있나요

- 담당자 9명의 카드를 클릭하면 역할, 행동지침, 바로 쓰는 프롬프트가 나옵니다
- **복사하기** 버튼을 누르면 프롬프트가 클립보드에 복사됩니다 → Claude, ChatGPT 등에 바로 붙여넣기
- 담당자별 지침을 `.md` 파일로 다운로드할 수 있습니다
- 검색창으로 원하는 담당자나 업무를 바로 찾을 수 있습니다
- 코딩 지식이 없는 제3자도 그대로 사용 가능합니다 (서버 불필요, 정적 사이트)

---

## GitHub Pages 배포 방법 (5분)

### 1. 저장소에 파일 업로드
이 폴더 안의 4개 파일을 저장소 루트에 그대로 올려주세요.
```
index.html
style.css
app.js
data.js
```

### 2. GitHub Pages 활성화
1. 저장소 페이지에서 **Settings** 클릭
2. 왼쪽 메뉴에서 **Pages** 클릭
3. **Source**를 `Deploy from a branch`로 설정
4. **Branch**를 `main` / `(root)`로 설정 후 **Save**
5. 1~2분 기다리면 상단에 배포된 주소가 표시됩니다

### 3. 완료
주소를 카카오톡, 블로그, 패들렛 등에 공유하면 누구나 바로 사용할 수 있습니다.

---

## 내용 수정 방법

담당자 정보나 프롬프트를 바꾸고 싶다면 **`data.js`** 파일 하나만 수정하면 됩니다.
HTML이나 CSS는 건드릴 필요 없습니다.

```js
{
  id: "blog",                    // 고유 ID (영문, 변경 금지)
  num: "02",                     // 번호
  name: "블로그 담당",            // 담당자 이름
  role: "네이버 · 티스토리 포스팅 작성",  // 한 줄 역할
  color: "#0F6E56",              // 카드 강조색
  bg: "#E1F5EE",                 // 카드 배경색
  tasks: ["블로그 초안", "..."],  // 태그 (카드에 표시)
  mission: "한 줄 소개 문구",
  guideline: `## 핵심 역할 ...`,  // 전체 행동지침 (마크다운)
  prompts: [
    { label: "버튼에 표시될 이름", text: "실제 복사될 프롬프트 내용" },
  ],
  collab: ["협업 담당자1", "협업 담당자2"],
}
```

새 담당자를 추가하려면 이 형식 그대로 객체 하나를 배열에 추가하면 됩니다.

---

## 기술 구성
- 순수 HTML / CSS / JavaScript (프레임워크 없음)
- 외부 라이브러리 없음 (구글 폰트 1개만 사용)
- 별도 백엔드·서버 불필요 — 정적 파일만으로 작동
- 모바일 반응형 지원
